Common Questions

Technical details about the Refactory hybrid engine.

How do Golden Snapshots prevent regressions?

Before any decomposition, Refactory scans the monolith and records every exported name and its typeof. After extraction, it compares the new modular surface against this snapshot. If a function was lost, renamed, or changed from a function to an object, the verifier catches it instantly.

What is Adaptive Compression?

Planning a 20,000 line decomposition would exceed most LLM context windows if we sent the full source. Refactory sends a compressed function map (metadata only) which is 10x smaller. The AI reasons about the map, and our deterministic engine performs the actual extraction using the original local source.

How does ast-grep integration work?

Refactory uses ast-grep for deep AST analysis. This allows us to identify "hot" routines, track cyclomatic complexity, and detect magic numbers or "HACK" comments with much higher precision than simple regex scanners.

Is it truly $0 token cost?

Mostly, yes. You need a free key (Groq or Gemini) for the PLAN step (deciding boundaries). The EXTRACT step for JS and Python handles the routine moves locally and deterministically. The AI still handles complex edge cases, but the bulk of the extraction costs nothing.

Which languages are supported?

JS/TS and Python have built-in deterministic preprocessors that handle the straightforward extractions with high syntax fidelity. Premium language packs (Go, Rust, Java, etc.) are planned for enterprise users. Other languages use LLM-based extraction with adaptive compression.
